V18: Medium FP32 Panel

V18 keeps V17's CuTe QR32 path and V13's safe double-precision panel for n=512, but tries a faster FP32 shared-memory panel for the medium benchmark shapes n=176 and n=352.

Dispatch

n = 32
CuTe DSL QR32, one warp per matrix, 32x32 tile and tau staged in shared memory.
n <= 352, batch >= 16
FP32 fused CUDA panel, then existing ATen WY trailing update.
n = 512, batch >= 16
V13 fused CUDA panel with double accumulation, preserving stress-test stability.
Large low-batch shapes
Fall back to torch.geqrf, which remains fastest for 1024/2048/4096.

Why It Should Help

V13's double-precision panel is robust but more expensive than needed for the dense medium benchmark shapes. V18 tests whether FP32 panel reductions can reduce latency for 176 and 352 without touching the numerically sensitive 512 stress cases.
