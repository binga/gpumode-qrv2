V17: CuTe QR32 + V13 Best Path

V17 promotes the successful V16 CuTe DSL QR32 experiment into the current best V13 solution. It keeps V13's fused double-accumulation panel and ATen trailing update for medium shapes, while replacing the small 32x32 path with a shared-memory CuTe DSL Householder kernel.

Dispatch

n = 32
CuTe DSL QR32, one warp per matrix, 32x32 tile and tau staged in shared memory.
n <= 512, batch >= 16
V13 fused CUDA panel with double accumulation, then existing ATen WY trailing update.
Large low-batch shapes
Fall back to torch.geqrf, which remains fastest for 1024/2048/4096.

Why It Should Help

The V16 probe showed the CuTe QR32 branch at 85.4us on B200, compared with roughly 134us for the prior CUDA small kernel. V17 isolates that improvement while preserving V13's best known behavior on every other shape.
