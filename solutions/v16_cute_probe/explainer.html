V16: CuTe DSL QR32

V16 proves that NVIDIA CuTe DSL works inside Popcorn's B200 runner and uses it for a real one-shape QR kernel: shared-memory Householder QR for n=32. Other shapes still fall back to baseline behavior in this probe file, so only the 32x32 timing should be treated as candidate signal.

Why This Exists

The existing CUDA small-matrix kernel was already fast at roughly 134us, but CuTe's shared-memory version cut that to 85.4us while passing all correctness tests. The same submission also keeps a tiny n=512 CuTe probe so future trailing-update work can build on a verified runtime path.

Execution Flow

1. Dispatch
n=32 attempts the CuTe QR32 kernel first.
2. QR32
One warp per matrix stages the 32x32 tile and tau in shared memory.
3. Probe
n=512 cases compile and launch a tiny CuTe kernel to prove runtime support.
4. Fallback
If any CuTe step fails, correctness falls back to torch.geqrf.

Expected Outcomes

  • Correctness: passed 22/22 on B200 with both CuTe QR32 and the n=512 probe active.
  • Performance: (20,32) improved to 85.4us, beating V13/V15's ~134us.
  • Next target: integrate QR32 into the V13 best path, then attack the high-batch n=512 trailing update.
