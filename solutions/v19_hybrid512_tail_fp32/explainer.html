V19: Hybrid 512 Tail Panels

V19 keeps V18's CuTe QR32 and medium FP32 panel improvements, then probes a conservative 512 optimization: tall early 512 panels still use V13's double accumulation, while smaller tail panels switch to the faster FP32 panel.

Dispatch

n = 32
CuTe DSL QR32, one warp per matrix, 32x32 tile and tau staged in shared memory.
n <= 352, batch >= 16
FP32 fused CUDA panel, then existing ATen WY trailing update.
n = 512, batch >= 16
Double accumulation while panel height is large; FP32 panel when the active height drops to 352 or below.
Large low-batch shapes
Fall back to torch.geqrf, which remains fastest for 1024/2048/4096.

Why It Should Help

The later 512 panels operate on much shorter active columns. V19 tests whether those tail panels can use FP32 reductions without violating the rank-deficient, band, rowscale, and near-collinear correctness checks.
